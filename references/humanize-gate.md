## 3. INV 16 — BAN_LEXICON (어휘 차단)

상세: `references/banwords-lexicon.md` (60+ 카탈로그 + 평문 변환표)

**핵심 7카테고리:**
1. AI 메타담화 어구 (결론적으로·요약하면 등)
2. AI 형식주의 부사 (본질적으로·구체적으로 등)
3. AI 접속 비대 (한편·게다가·나아가 등)
4. 영한 번역투 (~에 대한·~을 통해 등)
5. 형식명사 비대 (~것이다·~수 있다 등)
6. 작업라벨 구조어 (1축·2축·레이어·페이즈)
7. 형 시그니처 (장려, 강제 ✗)

**자체검사:** `scripts/humanize_check.py {산출파일}`

## 4. INV 17 — STYLE_NORTH_STAR (지향 좌표 10조)

산출물이 *지향*해야 할 문체 좌표. 한국 명문 산문(김훈·박완서·신영복·유시민) + 한국 광고PT(라텔 형 코퍼스) 융합.

| # | 조항 | 측정 |
|---|------|------|
| 1 | 평균 문장 길이 8~20자 | 어절 수 σ |
| 2 | 구체명사 비율 70%+ | 추상명사 (상황·현실·문제·의미·현상·관점) 회피 |
| 3 | 능동형 강제 | "되다·어지다·되었다·될 수 있다" 회피 |
| 4 | 한자어 50% 이하 | 고유어 우선 |
| 5 | 종결어미 다양화 | "-다" 단정형 3회 연속 금지 |
| 6 | 접속사 문단당 1회 이하 | "그리고·하지만·그래서" 절제 |
| 7 | 복수표지 "들" 회피 | "문제들이 많다" → "문제가 많다" |
| 8 | 시그니처 어휘 자연 주입 | 직접·그냥·왜·어떻게 (강제 ✗) |
| 9 | 수치 + 출처 박싱 | "3,200명 (KOSIS 2024)" |
| 10 | 단문 행갈이 호흡 | 한 문단 80자 초과 시 분절 |

**자체검사 절차:**
1. 산출물 모든 본문 문장에 1~10조 적용
2. 위반 3개 이상 = 재작성
3. 위반 1~2개 = 해당 부분만 보강

## 5. INV 18 — ANTI_BLOAT (30% 핵심 강제 압축)

형 명시 1번 증상 — "AI는 30%만 핵심, 70%는 쓰레기"의 정체. RLHF의 length bias + verbosity compensation이 원인.

기존 §C 3패스(형용사·연결어·중복) → **9패스 강화**:

| 패스 | 대상 | 효과 |
|---|------|------|
| 1 | 질문 재술·섹션 미리보기 ("이 섹션에서는~") | -5% |
| 2 | 서문·종결 표현 ("훌륭한 질문입니다·도움이 되셨길") | -5% |
| 3 | 헤징 캐스케이드 ("일반적으로 종종 보통" → 1개) | -8% |
| 4 | 동의어 나열 ("다양한·여러·각종" → 1개) | -7% |
| 5 | 예시 1개 제한 (3개 이상 → 1개) | -10% |
| 6 | 3항 강박 해체 (콤마 3개 → 핵심 1~2개) | -8% |
| 7 | "할 수 있다" 우회 정규화 ("X를 통해 Y할 수 있다" → "X하면 Y다") | -7% |
| 8 | 수식어 누적 제거 ("매우 중요하고 필수적이며 핵심적인" → "필수적인") | -5% |
| 9 | **산출 직전 70% 절단 게이트** (Strunk & White 자동화) | -강제 |

**9패스 합산 목표:** 50%+ 감축. 원본 1000자 → 산출 500자 이하.

**자체검사 질문:**
- "이 문장이 없으면 결론이 흔들리나?" NO → 삭제
- "이 단어가 같은 의미를 두 번 말하나?" YES → 1개 삭제
- "이 문단이 다음 문단과 같은 말을 하나?" YES → 통합 또는 삭제

## 6. 파이프라인 통합 (v3.0)

```
PREFLIGHT → §A 라우터 → §B 작성 (HERO + DOC_TYPE)
→ §C-NEW 9패스(ANTI_BLOAT, INV 18)
→ §D 6항 QC
→ §E 포맷 + CEW + 반대주장
→ §H STYLE_CHECK (INV 17 STYLE_NORTH_STAR 10조)
→ §I BAN_LEXICON grep (INV 16, scripts/humanize_check.py)
→ 적발시 평문변환·재출력 (루프 max 2회)
→ design-skill cascade
→ §J 작업라벨·HERO·어조 룰 스캔 (INV 13·14·15 기존)
→ 산출
```

## 7. 가타 (Gotchas) — v3.0 신규

| 함정 | 대응 |
|---|---|
| BAN 단어 발견 후 동의어로 부분치환 | 사고 자체가 AI식. 전수재작성만 통과 |
| STYLE_NORTH_STAR 10조 다 못 맞춰서 포기 | 핵심은 1·2·3·5·9. 나머지는 보강 |
| ANTI_BLOAT 70% 절단으로 정보 누락 | 절단은 *수식·반복*만. 사실·수치는 절대 보존 |
| 시그니처 어휘 강제 주입 → 부자연 | 빈도 가이드만. "직접" 무리하게 끼우지 말 것 |
| 형 코퍼스에 5회 등장한 "패러다임" 같은 예외 | 마케팅 업계어. ALLOW. 빈도 0 ≠ 절대 BAN |
| Burstiness 자체측정 토큰 비용 | LIGHT 모드는 사전 grep만. DEEP에서만 5회 재생성 |
