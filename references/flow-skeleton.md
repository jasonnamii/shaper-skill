## 2. 8단 기본 흐름

```
요약 → 접근 → 발견 → 전략 → 통찰 → 액션아이템 → 요약(말미) → 첨부
```

| 단 | 하는 일 | 분량 감각 |
|---|---|---|
| **1 요약** | 결론·핵심 제안 예고. 30초만 읽어도 전체가 잡힌다 | 짧게 |
| **2 접근** | 왜 이 문제를, 어떤 관점으로 보았나. 독자와 출발점을 맞춘다 | 짧게 |
| **3 발견** | 확인한 사실·수치·관찰. 해석 전의 재료 | 본문 |
| **4 전략** | 발견 위에서 무엇을 할 것인가. 제안의 몸통 | 본문 최대 |
| **5 통찰** | 발견과 전략을 관통하는 한 수. 스파인이 가장 선명해지는 자리 | 짧고 강하게 |
| **6 액션아이템** | 누가·무엇을·언제. 실행 단위 (§4 나열형 방식) | 본문 |
| **7 요약(말미)** | 결정 요청·다음 행동. 다 읽은 독자의 결론 | 짧게 |
| **8 첨부** | 본문 흐름을 끊는 근거·데이터·상세의 자리 | 자유 |

**두 요약의 역할 분리** — 서두 요약은 아직 안 읽은 독자에게 결론을 예고하고, 말미 요약은 다 읽은 독자에게 결정과 다음 행동을 요청한다. 역할이 다르므로 문장 하나도 재사용하지 않는다. 겹치면 말미를 다시 쓴다.

**첨부 = 오컴의 집행 장치** — 본문에서 깎는 것은 삭제가 아니라 강등이다. 흐름에 기여하지 못하지만 근거 가치가 있는 표·데이터·상세 설명은 첨부로 보내고 본문엔 참조 1줄만 남긴다. 첨부가 없는데 본문이 무겁다면 깎기를 안 한 것이다.

## 3. 변형 규칙 — 스켈레톤은 고정 틀이 아니다

8단은 출발점이고, 문서마다 통합·생략·확장해서 쓴다.

- **판정 질문:** "이 문서·이 독자에게 이 단이 일하는가?" 일 없는 단은 지운다. 접근이 자명하면 생략, 발견이 없으면(순수 제안서) 전략으로 직행, 실행 문서면 액션아이템을 확장.
- **통합:** 짧은 문서는 요약·전략·액션 3단까지 압축 가능. 발견+전략을 한 章으로 묶어도 된다.
- **순서:** 단의 순서 역행은 §4 배열 선택(미괄식)으로만 한다. 임의 뒤섞기는 변형이 아니라 붕괴.
- **doc-types와의 관계:** `doc-types.md`의 DOC_TYPE 골격(DELIVER·DIAGNOSE·EVALUATE·CONVERGE·BIZ_DOC)은 발견~전략 구간의 **본문 의무**(근거 컬럼·교차비교·논증 단위)로 흡수된다. 전체 배열은 본 문서가, 섹션 내부 의무는 doc-types가 정한다.

## 4. 세 배열 — 두괄식 · 미괄식 · 나열형

### 4-1. 두괄식 (기본)

8단 순서 그대로. 결론이 먼저 온다. 별도 조건 없이 모든 문서의 디폴트.

### 4-2. 미괄식 (클라이맥스)

**조건:** 아이디어·컨셉 자체가 상품인 기획안 — 컨셉이 마지막에 터져야 설득이 사는 경우에만. 보고서·진단서·실행 문서엔 쓰지 않는다.

- 서두 요약은 결론을 다 까지 않고 "무엇을 다루는 문서인지"와 판의 긴장만 건다.
- 접근→발견→전략이 컨셉을 향해 쌓이고, 통찰 자리에서 컨셉이 터진다(클라이맥스).
- 클라이맥스 이후의 액션아이템·말미 요약은 두괄식과 동일하게 닫는다.
- **금지:** 미괄식을 핑계로 한 뜸들이기. 빌드업의 각 단은 새 정보·새 근거여야 한다. 같은 말을 누적하는 빌드업은 클라이맥스가 아니라 반복 결함이다.

### 4-3. 나열형 (전체 조명 → 하나씩)

**조건:** 처음부터 끝까지 아이템 나열이 본질인 문서 — 라인업·실행 패키지·프로그램 목록·카탈로그. 액션아이템 단이 길어질 때도 이 방식을 쓴다.

```
[전체 조명]  한 화면(표·지도)에 전 아이템 — 독자가 지형을 먼저 갖는다
[하나씩]    아이템별 상세 — 같은 골격, 겹치지 않는 내용
```

- **전체 조명 먼저:** 아이템 × 한 줄 정의의 지도를 첫 화면에 둔다. 독자는 각 항목을 읽기 전에 전체 그림을 가진다.
- **그 다음 하나씩:** 모든 아이템이 같은 골격(이름 → 한 줄 정의 → 내용 → 실행)으로 흐른다. 골격의 반복은 리듬이고, 내용의 반복은 결함이다.
- 전체 조명에 쓴 한 줄 정의를 아이템 상세의 첫 문장으로 복붙하지 않는다 — 상세는 정의 다음 정보부터 시작한다.

## 5. 헤드라인 — 일하되 멋부리지 않는다

- 헤드라인만 통독하면 8단 변형 흐름이 복원되어야 한다 (`narr-gate.md` N1과 동일 권위).
- 헤드라인은 그 섹션이 하는 일을 말한다. 구조어휘(현황·분석·시사점·결론)도, 슬로건도 금지.
- **멋부리기 금지:** 선언적·철학적·공상적 헤드라인은 그 자리에서 재작성한다. 담담 톤 정본 = `hero-format.md` §5. 판정은 소리 내어 읽기 — 외침·카피 느낌이면 실패, 관찰·사실 진술 느낌이면 통과.

## 6. 기획안 전용 — 상대의 산에서 창의

기획안은 상대가 듣고 싶어하는 세계관 안에서 독창적이어야 한다.

- **상대 분석이 출발:** 접근 단에서 상대의 기존 사업·언어·우선순위를 먼저 잡는다. 제안은 상대 판에 붙는 매칭이 먼저고, 작명·컨셉의 화려함은 그 다음이다.
- **다른 산 금지:** 상대 관심 밖의 멋진 아이디어는 기획도 창의도 아니다. 독창성은 상대의 산 안에서 아무도 안 가본 길을 내는 것이다.
- **창의 안의 논리:** 독창적 제안일수록 발견→전략 인과가 더 촘촘해야 한다. 비약은 컨셉이 아니라 결함이다.
- **쉬움이 관문:** 상대가 한 번 읽고 이해하지 못하면 창의가 아니라 소음이다. 창의적인 문서일수록 문장은 더 평이해야 한다.

## 7. 반복·중복 — 흐름의 제1 결함

- 같은 판단·정보는 문서에 한 번. 허용 중복은 서두 요약·말미 요약 두 자리뿐 — 그마저 문장 재사용은 금지(§2 역할 분리).
- 표·도식에 넣은 정보를 산문으로 다시 풀지 않는다. 산문은 표가 못 하는 판단·연결만 한다 (`viz-fallback.md` §2).
- 단과 단 사이 브리지에서 직전 단을 요약하지 않는다 — 다음 단으로 가는 이유만 쓴다.
- 강조가 필요하면 반복이 아니라 논리 전개로 한다. 같은 주장을 두 번 쓰고 싶어지면 두 번째 자리에 새 근거를 넣는다.

## 8. 고정과 자유 — 이 가이드의 강도 설계

가이드가 빡빡할수록 문서가 닮아간다. 그래서 강제는 결함을 막는 데까지만 걸고, 유니크함이 사는 자리는 비워 둔다.

| 고정 (어기면 결함) | 자유 (문서마다 설계) |
|---|---|
| 반복·중복 금지 (§7) | 단계 통합·생략·확장 (§3) |
| 헤드라인 멋부리기 금지 (§5) | 배열 선택 — 두괄·미괄·나열 (§4) |
| 미검증 수치·사실 단정 금지 | 분량 배분·밀도 리듬 |
| 상대 세계관 이탈 금지 (§6) | 시각 장치의 종류·위치 (`viz-fallback.md`) |
| 본문 과적 — 깎은 것은 첨부로 (§2) | 어휘·리듬·사례 선택 — 유니크함이 사는 자리 |

자유 칸을 채우는 것도 설계다. 매 문서에서 "이 문서만의 자유 선택이 무엇인가"를 한 번 자문하고, 답이 없으면 평균 문서가 나온 것이다.

## 9. 자가검사 (산출 직전 1회)

1. 헤드라인만 통독 — 8단 변형 흐름이 복원되는가.
2. 서두·말미 요약 — 문장 하나라도 겹치는가. 겹치면 말미 재작성.
3. 일 안 하는 단이 있는가 — 지우거나 첨부로 강등.
4. 미괄식이면 — 빌드업 각 단이 새 정보인가.
5. 나열형이면 — 전체 조명 1화면이 있는가, 정의 복붙이 없는가.
6. 기획안이면 — 상대의 산 위에 있는가, 인과 비약이 없는가.

## 10. Gotchas

| 함정 | 대응 |
|---|---|
| 말미 요약이 서두 요약의 재방송 | 말미 = 결정·다음 행동으로 재작성 |
| 미괄식 빌드업이 동어 누적 | 단마다 새 정보 — 없으면 두괄식 전환 |
| 나열형에서 전체 조명 생략 | 첫 화면에 아이템 지도 의무 |
| 깎은 내용을 그냥 삭제 | 근거 가치 있으면 첨부로 강등 |
| 8단을 전부 채우려는 강박 | 일 없는 단은 지운다 — 변형이 기본 |
| 컨셉이 화려한데 상대 판과 무관 | §6 다른 산 — 접근 단부터 재설계 |
