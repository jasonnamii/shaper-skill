## 2. §A-5 SPINE 응결 프로세스

§A-4 LV 결정 직후·§B-PRE 진입 *전*. MODE_M(500~2000자)·MODE_L(>2000자) 강제.

### 2-1. 프로세스 3단

```
1. Claude가 요청 맥락에서 스파인 1줄(≤30자) 제안
2. 형 컴펌 1회 — `🎯 스파인 1줄: "______". OK·수정·재제안 [O/S/R]`
3. OK = 박제 → §B-PRE 진입 / 수정 = 형 직접 입력 / 재제안 = Claude 다른 안 1회 (max 2회)
```

### 2-2. 스파인 품질 기준

| 기준 | YES | NO |
|---|---|---|
| 길이 | ≤30자 | 30자+ → 압축 재제안 |
| 단언형 | "X가 답이다·X는 Y다" | 의문·열거·평이 진술 |
| 1주장 | 1개 핵심 명제 | 다중 명제·접속절 |
| 헤드라인 역추론 | 모든 헤드라인이 이 1줄의 변주 | 헤드라인이 구조어휘 |

### 2-3. SCOPE_OUT (면제)

- MODE_S (≤500자)
- 단답·핑퐁·실행보고
- 형 명시 SCOPE_OUT 토큰 (UP 본문 면제 일관)
- 단순 확인성 응답

## 3. §B-NARR 4종 룰

§B-PRE 통과 후·§B-구조 작성 전 강제. 작성자(persona-corpus §2 본업) + 편집자(이오덕 디폴트) 듀얼 캐스팅.

### 3-1. N1 — 스파인 응결 (헤드라인=백본 정합)

**적용:** 모든 헤드라인 입력 시점.

**자가검열:** "이 헤드라인 보고 §A-5 박제 스파인을 역추론할 수 있나?"
- YES → 진행
- NO → 헤드라인 재작성

**메커니즘:**
- 모든 헤드라인 = 스파인의 변주
- 헤드라인-only 통독으로 스파인 직관 가능
- 구조어휘(현황·분석·시사점·결론) BAN

**❌ WRONG / ✅ CORRECT:**
```
❌ 스파인 = "정원이 답이다"
   헤드라인 = "1. 현황 / 2. 분석 / 3. 시사점 / 4. 결론"
   (구조어휘만·스파인 역추론 ✗)

✅ 스파인 = "정원이 답이다"
   헤드라인 = "1. 양평DNA는 셋이다 / 2. 정원이 가장 강하다 / 3. 정원으로 +12% / 4. 정원에 베팅한다"
   (모두 스파인 변주·역추론 ✓)
```

### 3-2. N2 — 에디터 페어 (작성자+편집자 분리)

**적용:** 섹션(헤드라인 1개 단위) 종료 시.

**프로세스:**
1. 작성자(persona-corpus §2 매핑) 한 섹션 작성 완료
2. 편집자 페르소나(이오덕 디폴트·자갈제거·번역투박멸) 진입
3. 편집자 4문:
   - "이 섹션이 스파인을 더 뾰족하게 받쳐주나?"
   - "앞 섹션과 중복? (어휘·문장구조 양쪽)"
   - "헤드라인만 봐도 백본이 보이나?"
   - "30% 잘라도 사는가? 자갈은?"
4. 1개라도 NO → 작성자 재진입

**편집자 페르소나 캐스팅:**

| DOC_TYPE | 작성자 | 편집자 디폴트 |
|---|---|---|
| DELIVER | 강원국 | 이오덕 |
| DIAGNOSE | 이오덕 | 강원국 (편집자 충돌 시) |
| EVALUATE | 이오덕 | 강원국 (편집자 충돌 시) |
| CONVERGE | 유시민 | 이오덕 |
| HERO | 김훈 | 이오덕 |

**충돌 해소:** 작성자=이오덕인 경우(DIAGNOSE·EVALUATE) → 편집자는 강원국으로 자동 전환. 같은 페르소나 페어 ✗.

**근거:** Pound가 Eliot Waste Land 50% 잘라냄 (1922). NYT writer-editor-copy editor 3단. Pixar Brain Trust. 외부 시선 = 자기검열 강도 2배.

### 3-3. N3 — 반박자 동석 (사전 자기비판)

**적용:** 모든 핵심 주장 입력 시점.

**프로세스:**
1. 핵심 주장 입력 → *내적* 자문 1줄 "반박자 왈: ___"
2. 반박을 1문장으로 명시
3. 통과한 주장만 본문 진입·통과 못한 주장 폐기

**출력 원칙:**
- 자문은 *내적*만·본문에 출력 ✗
- 통과 못한 주장 = 본문에서 폐기
- 통과한 큰 반박 = INV 12 반대주장 박스로 사후 표시 (MODE_M·L)

**근거:** Klein Premortem (2007) — 결정 전 실패 가정 회고. 카너먼 System 2 강제 가동.

**❌ WRONG / ✅ CORRECT:**
```
❌ 본문에 "반박자 왈: 기능 차별이 우선 아닌가?" 직접 출력 → N3 위반·자문은 내적
✅ 내적 자문 통과 후 본문에는 "기능보다 소유 구조다 (반박: 기능 차별이 우선 — 통계 3종으로 기각)"
   또는 통과 못한 주장은 본문 ZERO
```

### 3-4. N4 — CEWA 진화 (Analogy 슬롯 의무)

**적용:** 모든 주장 작성 시점. INV 10 CEW의 진화형.

**메커니즘:**
- C(Claim) + E(Evidence) + W(Warrant) + **A(Analogy)** 4슬롯
- 주장당 유사 성공사례 1+ 동반 의무
- "X는 Y와 같다·Y는 이렇게 됐다" 구조

**사례풀 우선순위:**
1. 형 코퍼스 (jason-corpus-examples.md)
2. persona-corpus 명저자 인용 사례
3. 6수사덱 (`rhetoric-deck.md`) Analogy 카드
4. 외부 1차 출처 (검증된 케이스만)

**근거:** 카너먼 System 1 — 유사사례 1개가 통계 100개보다 강함. Heath Made to Stick — Concrete·Stories·Credible.

**SCOPE_OUT (A 슬롯 면제):**
- 단순 사실 진술 (수치·날짜·고유명사)
- 표·코드·인용박스
- HERO 첫줄 (압축 우선)

## 4. 거시 4문 (§B-NARR 진입 직전 자기검열)

> 1. "스파인 1줄 응결됐나?" (§A-5 박제 확인)
> 2. "헤드라인 통독으로 스파인 직관 가능?" (N1)
> 3. "핵심 주장 반박자 통과?" (N3)
> 4. "주장당 아날로지 1+?" (N4)

4문 모두 YES → §B-구조 진입. 1개라도 NO → 재진입.

## 6. 충돌 해소

| 충돌 | 해소 |
|---|---|
| §B-NARR vs §B-PRE | 둘 다 사전강제. 순서 = §B-PRE 먼저(미시) → §B-NARR(거시) |
| N2 작성자=편집자 동일 페르소나 | 편집자 자동 전환 (이오덕 ↔ 강원국) |
| N3 자문이 본문 출력 | 즉시 폐기·재작성 (자문은 내적만) |
| N4 사례 부재 | 형 코퍼스 → persona-corpus → 6수사덱 → 외부 순. 외부도 부재 = 주장 자체 폐기 |
| INV 22 격식 vs N3 반박자 | 격식 우선·반박자도 합쇼체 |

## 7. cleanup 축15와의 직교성

| | shaper §B-NARR (사전) | cleanup 축15 (사후) |
|---|---|---|
| 시점 | 작성 중 | 송출 직전 |
| 권한 | 강제·재작성 | 검출·경고만 |
| 콘텐츠 | 변경 가능 | 불변 (Rule 5) |

**원칙:** cleanup 축15가 적발하면 shaper §B-NARR로 *되돌려* 재작성. cleanup이 직접 서사 재구성 ✗.

## 8. Gotchas

| 함정 | 대응 |
|---|---|
| §A-5 SPINE 스킵하고 §B 진입 | 거시 4문 1번 자동 FAIL → §A-5 재진입 |
| MODE_S에 §B-NARR 강제 | SCOPE_OUT 위반·면제 영역 |
| N3 자문을 본문 출력 | 자문은 *내적*만. 위반 시 즉시 재작성 |
| N4 외부 사례 검증 없이 인용 | 1차 출처 신뢰 시에만 |
| 작성자=편집자 동일 페르소나 | 편집자 자동 전환 |
| §B-NARR 통과 없이 §B-구조 진입 | INV 27 위반 = TOP-LEVEL FAIL → §B-NARR 재진입 |
