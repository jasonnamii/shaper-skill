# NO_WORK_LABEL — 본질보호 게이트 (v2.5 신규, INV 13)

> 상위 정본: `references/writing-standard.md` §3 표현금지. 본 문서는 해당 섹션의 세부 백본.

산출물·대화 모두에 적용. 작업 라벨 ZERO. 1만 페이지에 1단어라도 = FAIL → 전수 재작성.

이 문서는 SKILL.md INV 13의 단일 권위. validate.py 사전·패턴은 이 본질의 *보조*일 뿐.

---

## 1. 본질 5줄

| | |
|---|---|
| **RULE** | 산출물·대화 = 인간 언어. 작업 라벨 ZERO. (1만 페이지 1단어 = FAIL) |
| **판정** | "이 단어, 이 대화 밖 사람이 사전 없이 읽을 수 있나?" — NO → 작업 라벨 → 금지 |
| **ALLOW** | 업계 전문용어 (BEP·KPI·MECE·MVP 등) · 고유명사 · 법조문 |
| **CONVERT** | 라벨 발견 → 실명·평문 풀어쓰기 (예: Y2 → "2년차" / 4축 → 실제 4개 이름 / C:E:W: → 평문) |
| **SELF_CHECK** | 산출 직전 자체 스캔. 1개라도 발견 = 차단·재작성 |

다른 모든 규정은 이 5줄의 보조다.

---

## 2. 판정 — 1줄 질문

산출물 모든 단어에 적용:

> **"이 단어, 이 대화 밖 사람이 사전 없이 읽을 수 있나?"**

- **YES** → 통과
- **NO** → 작업 라벨 → 차단·실명 풀어쓰기

판정질문 1개가 본질. 사전·패턴은 보조. 사전에 누락된 단어가 새 나와도 판정질문은 작동한다.

---

## 3. ALLOW 화이트리스트

다음은 작업 라벨처럼 보여도 통과:

| 분류 | 예 | 통과 사유 |
|------|-----|---------|
| 업계 전문용어 | BEP, KPI, MECE, MVP, ROI, ROAS, CAC, LTV, OKR, MOU, NDA, IPO | 해당 업계인이 사전 없이 읽음 |
| 고유명사 | Glyndebourne, KOSIS, Pemberton, 옥스포드, 자라섬 | 사전이 아니라 검색으로 확인되는 실체 |
| 법조문·약어 | 공연법 2024, GDPR, 개정안 §3 | 법률 인용 |
| 시장·금융 용어 | 시드라운드, 시리즈A, 텀시트, 캡테이블 | 스타트업 업계 표준 |
| 측정·통계 | p-value, 신뢰구간, 표본수, 가설검증 | 학술·실무 표준 |

**판정 보조 질문**: "이 단어를 위키피디아·구글 한 번이면 누구나 정확히 같은 뜻으로 이해하나?" YES → 통과.

## 4. CONVERT — 라벨 → 실명·평문

라벨식 표현은 라벨로 바꾸지 말고 **실명·평문으로 풀어쓴다**.

| 라벨 (X) | 실명·평문 (O) |
|---------|--------------|
| Y1 / Y2 / Y3 | 1년차 / 2년차 / 3년차 |
| 4축 (정원·물·문학·휴식) | "정원·물·문학·휴식 네 가지" |
| 3트랙 (살롱·교육·메가) | "살롱·교육·메가 세 사업" |
| 마지막 레이어 | 마지막 결 / 마지막 겹 |
| 흑자축 | 흑자 사업 / 흑자 항목 |
| Phase 1 | 1단계 / 첫 단계 |
| C: 양평 음악 빈자리 | (평문) "양평에는 음악 슬롯이 비어있다" |
| C+E+W 한 줄 | 주장·증거·근거를 자연 문장 한 줄로 |
| Pin↔Body 매핑 | (섹션 자체 삭제 — 사용자 무관한 작업물) |
| 압정 1:4:5 | (메타 자체 삭제) |
| CONVERGE MODE_L | (메타 자체 삭제) |
| DEEP 4항 QC | (메타 자체 삭제) |
| 페이퍼엔진 통과 | (메타 자체 삭제) |
| INV 13 발동 | (메타 자체 삭제) |

**핵심 원칙 두 가지**

1. **라벨을 라벨로 치환하지 말라** — 축→다른 추상명사는 또 다른 라벨이다. 진짜 4개 이름을 풀어쓴다.
2. **메타 표기는 삭제** — "v3.1 페이퍼엔진 통과", "압정 1:4:5", "DEEP 4항 QC" 같은 자기참조 메타블록은 사용자에게 의미 0. 그냥 삭제한다.

## 5. SELF_CHECK — 산출 직전 자체 스캔

**1차 게이트 — LLM 자체 판정 (본질)**

산출물 모든 단어를 훑으며 판정질문을 적용. 1개라도 NO → 차단 → 전수 재작성.

**2차 게이트 — 스캐너 (보조)**

```bash
python scripts/validate.py --check-output {산출파일경로}
```

- 사전·패턴 매칭으로 자주 새는 단어를 1차 검출
- 화이트리스트 통과 단어는 OK
- 적발 1+ → 전수 재작성 → 재스캔. 2회차 적발 → STOP+사용자 보고

**부분치환 금지**

적발 단어만 바꾸지 말 것. 라벨식 사고가 본문 구조 전체에 박혀있어 부분치환만으론 자연어 글이 되지 않는다. **전수 재작성**으로 사고 자체를 바꾼다.

## 8. 우선순위 — NO_WORK_LABEL vs 다른 규칙

NO_WORK_LABEL은 **최상위**. 다른 규칙(Pin↔Body 매핑 표 의무, CEW 한 줄 등)이 라벨 단어를 요구하면 그 다른 규칙을 자연어로 풀어쓴다. 라벨이 통과하는 일은 없다.

예:
- "Pin↔Body 매핑 표 의무" → 자연어 "결론과 근거 연결" 또는 매핑 자체를 본문 구성으로 흡수 (별도 표 ✗)
- "CEW 한 줄" → 주장·증거·근거를 자연 문장 한 줄로
- "DOC_TYPE 분기 표기" → 산출물 메타에 표기하지 않음. 내부 라우팅만
