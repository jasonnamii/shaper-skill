| BAN | 변환 |
|---|---|
| 결론적으로·요약하면·정리하면·요컨대·한마디로 | (삭제·결론은 첫 줄에) |
| 라고 할 수 있다·라고 볼 수 있다·할 수 있다고 본다 | ~이다·~한다 |
| 주목할 만한·강조할 필요가 있다·다시 한번 강조하면 | (삭제) |
| 박제·박다·박혀·박힌·새기다 | 저장·기록·고정·넣다·두다·적다 |
| 혁신적·차세대·포용적·전사적·윈윈·유기적·역동적 | (구체 사실) |
| 다각도로·다층적으로·전례 없는·지속 가능한·차별화된 | (사실 직시) |
| 솔루션 기반·전략적 파트너·고객 중심·체계적이고·필수적인·선도적인 | (목적·실명) |

| 사용자 친화적·직관적이고·최적화된·사용자 경험을 향상 | (구체 사실) |

### 2-2. ADV_BAN (형식주의 부사 + KIWI 강조부사)
| BAN | 변환 |
|---|---|
| 본질적으로·근본적으로·구체적으로·전반적으로·점진적으로 | (실명·사실) |
| 효과적으로·효율적으로·전략적으로·혁신적으로·지속적으로 | (사실로) |
| 매우·너무나·대단히·굉장히·엄청나게·무척 | (삭제) |
| 천천히 (걷는다) | 어슬렁거렸다·빈둥거렸다 (강한 동사) |

**ALLOW (형 시그니처):** 정말·확실히·진짜·딱·솔직히·사실·근데·그냥·왜·어떻게

### 2-3. LABEL_BAN (작업라벨 + UP 작업프레임)
| BAN | 변환 |
|---|---|
| 1축·2축·3축·N축·1레이어·N레이어·1트랙·N트랙 | 실명 (예: "정원·물·문학·휴식 네 가지") |
| Phase 1·Phase N·Layer 1·Layer N·페이즈·레이어 | "1단계·2단계" 또는 실명 |
| 축별·다축·수렴축·차원이·차원에서·영역에서·관점에서·측면에서·종류로·범주로·항목으로·구분으로 | (실제 항목명 평문 열거) |
| P1·P2·MUST·SHOULD·MAY·SCOPE_OUT·SCOPE_IN·LIGHT·DEEP·TURBO | "1단계·반드시·권장·선택" |
| 인물 동학·조직 동학·경력 DNA·조직 DNA | 이력 패턴·본질 ("DNA" 단독은 ALLOW) |
| 자체검사·자체점검 | (침묵 수행) |
| 숫자카운터: `[0-9]+ + (축\|건\|대\|종\|개\|차원\|영역\|항목\|측면\|관점\|종류\|범주\|갈래\|방면\|국면\|요소\|구분)` | 실명 열거 |
| 갈래단독금지: `갈래` (단독 사용 일체) | 트랙·줄기·길·방향·경로 중 맥락 맞는 실명 |

**허용 평문 명사 (형 시그니처):** 이유·기준·핵심·방식·차이·요소·역할·방법·조건

| ~로 인하여·~로 인해·~함에 있어 | ~로·~때문에·~할 때 |

| ~수 있다고 본다·~할 필요가 있다·~하는 것이 중요하다 | ~할 수 있다·~해야 한다·~가 중요하다 |

| ~하는 것이다·~하는 것은·~한다는 것은 | ~한다·~함은·~한다는 점은 |

### 2-9. ABSTRACT_BAN (추상명사 본문 침투 차단·🆕 v4.3)

**RULE:** DELIVER·CONVERGE·DIAGNOSE·EVALUATE 모드 본문에서 추상명사 BAN. UTTER 모드(박웅현·매니페스토·카피·슬로건)만 ALLOW.

**SCOPE:** 본문 (Headline·Lead·Body·Pin↔Body). 카피 블록·슬로건 블록·매니페스토 블록은 면제 (block 단위 명시 면제만).

| BAN (추상명사) | 본문 변환 (일상어) |
|---|---|
| 주권·주권회복·자율성·자기결정권 | 스스로 정한다·내가 고른다·남이 못 정한다 |
| 자산·디지털자산·핵심자산·전략자산 | 가진 것·내 작업·쌓인 것 |
| OS·플랫폼·인프라·아키텍처·생태계 | 일하는 곳·만나는 곳·쓰는 도구·움직이는 판 |
| 시그니처·아이덴티티·페르소나·정체성 | 내 색·내 방식·나만의 것 |
| 구축·재구축·확립·정립·수립 | 만든다·다시 만든다·세운다 |
| 회복·복원·환원·반환·되찾음 | 다시 가진다·돌려받는다 |
| 진화·승화·고도화·혁신·도약 | 바뀐다·더 좋아진다·다음 단계로 간다 |
| 본질·정수·핵·코어 (단독 사용) | 진짜·중심·뿌리 |
| 가시성·존재감·임팩트·파급력 | 보인다·있다·퍼진다 |
| 패러다임·프레임워크·방법론 (단독 비유 ✗) | 방식·틀·길 |

**FAIL 조건 (본문):**
1. 위 추상명사 본문 등장 시 → **첫 등장 시 일상어 정의 1줄 의무**. 정의 없이 반복 = FAIL
2. 1문장에 추상명사 2+ 동시 출현 = FAIL (예: "디지털 페르소나 자산" / "알고리즘 주권 회복")
3. 추상명사 합성 (영문+한자 결합·한자+한자 결합) = FAIL (예: "Algorithm Sovereignty" 본문 노출, "자기 구축형 창작자 OS")

**자가합리화 차단 (UP, v4.3.1 — 5종):**
- "컨셉이니 예외" ✗
- "외래어 아니니 OK" ✗
- "마케팅 업계어니 OK" ✗
- "박웅현이 쓸 법하니 ALLOW" ✗ (UTTER 모드 명시 ✗ = BAN)
- **🆕 "내 검증 끝났으니 OK" ✗** — 화이트리스트 누락·section ID 빠짐·grep 범위 축소로 인한 false-clean. 검증 = `humanize_check.py scan_abstract()` 자동 grep 통과만 인정. 수동 grep ✗.

**UTTER 면제 매커니즘:** UTTER 모드로 호출된 블록만 ALLOW. 같은 문서 안에서도 본문(DELIVER) 블록 ✗ / 카피 블록(UTTER) ○. block-level 명시 면제만 인정.

**ALLOW (업계 표준·예외):**
- 마케팅 업계어 (lexicon-ban §3 그대로): 패러다임·최적화·플랫폼·콘텐츠·캠페인·브랜드·타겟팅·다이렉트·퍼포먼스
- 고유명사·법률용어
- 단, 본문 첫 등장 시 일상어 정의 1줄 동반

**❌ WRONG vs ✅ CORRECT:**

```
❌ 본문: "C8 = 자기 구축형 창작자 OS. 디지털 페르소나 자산을 통한 알고리즘 주권 회복."
   → 추상명사 5개·정의 0개·1문장 합성 폭주
✅ 본문: "C8은 창작자가 일하는 곳입니다. 작업물·이력·연결을 한 곳에 모읍니다.
        SNS 알고리즘이 자른 자기를, 다시 묶습니다."
   → 일상어. 추상명사 ✗.
✅ 카피 블록(UTTER): "알고리즘 주권 회복" ALLOW (UTTER 명시·면제)
```
